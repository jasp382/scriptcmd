## 1. Introdução ao Ambiente Linux e ao Ubuntu

Nesta disciplina iremos trabalhar num ambiente **Linux**, mais concretamente com a distribuição **Ubuntu 24.04 LTS**. Isto está alinhado com o ambiente tecnológico apresentado na unidade curricular, onde é indicado que os exercícios práticos serão realizados em Ubuntu e que este constitui o sistema base da componente prática.

<h3>Porquê usar Linux e Ubuntu nesta disciplina?</h3>
Existem vários motivos:

- **O curso está baseado em Bash**, a principal shell do ecossistema Linux. O bash é a ferramenta essencial para a navegação, automação e scripting apresentados na disciplina.  
    _→ O programa da disciplina destaca a utilização da linha de comandos, comandos básicos em bash e criação de scripts._
    
- **Ubuntu é um sistema operativo livre**, gratuito, aberto e amplamente utilizado na indústria, ensino e investigação.
    
- **Ferramentas como SSH, Git, Python e ferramentas de automação funcionam de forma mais simples e natural em Linux**.
    
- **Ambiente unificado**: independentemente da máquina que cada formando tem, podemos garantir que todos trabalham com o mesmo tipo de sistema.

## 3. O que é uma Máquina Virtual?

Uma **máquina virtual (VM)** é um software que simula um computador completo dentro de outro computador.  
Permite executar um sistema operativo (por exemplo, Ubuntu) como se fosse uma aplicação dentro do Windows ou macOS.

Vantagens:

- Permite usar Linux mesmo sem alterar o sistema principal.
- Ideal para isolar ambientes.
- Boa solução para máquinas mais antigas ou para quem prefere separação total entre sistemas.

Desvantagens:

- **Menos eficiente**: consome mais memória e CPU.
- **Integração limitada** com o sistema anfitrião.

Neste contexto, o WSL surge como uma tecnologia semelhante que suplanta algumas das desvantagens das máquinas virtuais clássicas. O **WSL (Windows Subsystem for Linux)** é uma tecnologia da Microsoft que permite executar um ambiente Linux diretamente dentro do Windows, **sem recorrer a uma máquina virtual tradicional**. Embora tecnicamente utilize componentes de virtualização, o WSL não cria um sistema completo isolado como fazem o VmWare ou o VirtualBox. Em vez disso, integra o núcleo Linux (no WSL2) de forma muito mais leve e profunda com o sistema operativo Windows.

Isto resolve várias limitações associadas às máquinas virtuais típicas, tais como o consumo elevado de memória, a menor performance em operações de disco e a fraca integração com o sistema anfitrião. O WSL permite, por exemplo, abrir pastas Linux diretamente no Explorador de Ficheiros, aceder a ficheiros Windows a partir do Ubuntu, partilhar portas de rede sem configurações adicionais e utilizar o Visual Studio Code nativamente dentro do ambiente Linux.  
Por este motivo, o WSL oferece uma experiência híbrida: tem a eficiência e integração de um sistema nativo, mas ao mesmo tempo fornece um ambiente Linux completo e compatível — sendo, por isso, **a opção mais recomendada para quem utiliza Windows** e precisa de trabalhar com bash, ferramentas de linha de comandos e Python num ambiente pedagógico ou profissional.

## 4. Alternativas para correr Ubuntu em Windows ou macOS

### 4.1. Em Windows

Existem duas opções principais:

#### ✅ **WSL (Windows Subsystem for Linux)** — Recomendado

_É a solução mais eficiente e cómoda._  
Esta metodologia é referida como ambiente de uso recomendado na disciplina, uma vez que os alunos trabalharão com Ubuntu e bash.

Vantagens:

- Excelente desempenho.
- Integração total com o Windows.
- Permite abrir pastas do Ubuntu diretamente no VS Code.
- Menor consumo de recursos.

#### ✅ Máquinas virtuais (VmWare Workstation Pro)

Opção alternativa para quem não puder usar WSL.

#### ✅ Multipass

Ferramenta da Canonical para criar VMs Ubuntu de forma rápida.  
É mais leve que VmWare, mas menos integrado que o WSL.
É também uma alternativa às duas anteriores.

---

### 4.2. Em macOS

As alternativas são:

#### ✅ VmWare Fusion (recomendado pelo curso)

Opção indicada para criar máquinas virtuais em MacOS.
#### ✅ Multipass

Funciona de forma semelhante ao Windows, criando pequenas VMs Ubuntu.

## 5. Comparação resumida entre WSL e Máquina Virtual

|Critério|WSL|Máquina Virtual|
|---|---|---|
|Desempenho|Muito elevado|Médio|
|Integração com host|Excelente|Moderada|
|Uso de recursos|Baixo|Alto|
|Facilidade de instalação|Muito simples|Média|
|Ideal para|Programação e terminal|Ambientes isolados ou específicos|

## 7. Instalação do Software necessário

Ao longo do curso vamos usar vários pacotes de software que temos de instalar de seguida:  
**Python, GIT, JupyterLab e vários Python Packages**.

Seguem os passos detalhados.

---

### 7.1. Instalar Git e clonar repositório da disciplina

```Bash
sudo apt install -y git

# Configuracao inicial
git config --global user.name "O Seu Nome"
git config --global user.email "O seu Email"

# Clonar repositório da disciplina
cd ~
git clone https://github.com/jasp382/scriptcmd.git
```


### 7.2. Instalar Python e ferramentas associadas

```Bash

sudo apt update && sudo apt upgrade -y
sudo apt install -y python3 python3-pip build-essential
sudo apt install python3-dev python3-virtualenv python3-venv virtualenvwrapper -y

echo "export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3" | sudo tee --append ~/.bashrc

echo "export WORKON_HOME=~/.virtualenvs" | sudo tee --append ~/.bashrc

echo "export VIRTUALENVWRAPPER_VIRTUALENV=/usr/bin/virtualenv" | sudo tee --append ~/.bashrc

echo "source /usr/share/virtualenvwrapper/virtualenvwrapper.sh" | sudo tee --append ~/.bashrc

source ~/.bashrc

```

Verificar:

```Bash
python3 --version  

pip3 --version 
```


---

### 7.3. Criar ambiente virtual Python

```Bash
# Create new virtual env

mkvirtualenv gs --system-site-packages

# Activate new environment
workon gs

# Install Python Packages
pip install --upgrade pip
cd ~/scriptcmd && pip install -r requirements.txt
```
